# Web3 Wiring (実体化) 仕様書

## 概要

ゲーム終了 → エージェント生成パイプラインの「Web3 部分」を文字列連結から実 SDK 呼び出しに置き換える。具体的には、0G Galileo testnet に iNFT を mint、Sepolia ENS subname を発行、Sepolia Uniswap で初取引を実行する。これにより 0G / ENS / Uniswap の 3 賞金トラックを cosmetic から meaningful な統合に引き上げる。

## ユーザーストーリー

- ハッカソンの審判として、demo 動画を見て「これは本当に on-chain で動いている」と一目で確認したい
- プレイヤーとして、60 秒プレイ後に自分のエージェントが explorer / ENS resolver / Uniswap で実際に履歴を残しているのを見たい
- 0G prize 審査員として、project page から 1 クリックで minted iNFT の 0G chain explorer link に飛べる
- ENS prize 審査員として、subname を sepolia.app.ens.domains で解決し、text record の検証可能な credential を見れる

## 受け入れ基準

- [ ] iNFT が 0G Galileo testnet で mint される (explorer link で確認可能)
- [ ] play log JSON が 0G Storage に put され、その CID が iNFT メタデータに埋まる
- [ ] Sepolia ENS で `{handle}.gradiusweb3.eth` subname が発行される (sepolia.app.ens.domains で解決可能)
- [ ] subname の text records に `combat-power` / `archetype` / `design-hash` が書かれる
- [ ] エージェント誕生後、ボタン 1 つで Sepolia Uniswap v3 swap が実行できる
- [ ] AgentDashboard に 4 つのリンクが追加: 0G iNFT explorer / ENS resolver / Uniswap tx hash / 0G storage CID
- [ ] FEEDBACK.md に Uniswap API DX learnings (実装中の摩擦) を追記
- [ ] 失敗ケース (RPC 切断 / wallet 拒否 / faucet 切れ) で UX が壊れない

## 非機能要件

- **セキュリティ**: 秘密鍵をフロントに置かない (wagmi の connected wallet で署名)。play_log 由来の deterministic wallet は表示用 placeholder として残すが、tx 署名はブラウザウォレットに委譲。
- **パフォーマンス**: ゲーム本体の 60fps を阻害しない (Web3 呼び出しは forge 完了後の post-processing)。
- **アクセシビリティ**: 失敗時の error message が日本語/英語で読める。tx 進行中は loading 状態を明示。
- **フェイルセーフ**: 0G / ENS / Uniswap いずれかが失敗してもエージェント birth 自体は完了する。各ステップを `Promise.allSettled` で並列実行し、失敗は表示するが他をブロックしない。
- **冪等性**: 同じ play log で再 mint しても重複 iNFT が出ないよう、tokenId を `keccak(playLogHash)` から導出。

## 技術設計

### コントラクト書き換え

1. `contracts/src/AgentForgeINFT.sol` を **ERC-721 + tokenURI 準拠** に書き換え:
   - OpenZeppelin の `ERC721` を base に
   - `tokenURI` は data URI (`data:application/json;base64,...`) で `role` / `policy` / `playLogHash` / `storageCID` / `archetype` / `combat-power` を返す
   - 0G Galileo にデプロイ (foundry の `Deploy.s.sol` に Galileo network 追加)
2. `contracts/src/AgentForgeSubnameRegistry.sol` は **削除** (本物 ENS を使うため)。ENS NameWrapper の `setSubnodeRecord` を直接呼ぶ。

### SDK / API 選定

- **0G Storage**: `@0glabs/0g-ts-sdk` (browser bundle) で put/get
- **ENS**: `viem` の `writeContract` で ENS NameWrapper / Resolver を直接叩く (専用 SDK 不要)
- **Uniswap**: `@uniswap/v3-sdk` + Sepolia v3 router (`SwapRouter02`)

### Forge → On-chain フロー

```
game over (BirthArcade onComplete)
  ↓
forge.ts: createAgentBirthDraft(playLog) [既存] — pure
  ↓
new module: packages/frontend/src/web3/forge-onchain.ts
  Promise.allSettled([
    step0gStorage.putPlayLog(playLog) → CID,
    step0gMint.mintINft(walletClient, profile, policy, playLogHash, CID) → tokenId,
    stepEnsRegister.subname(walletClient, handle, owner, textRecords) → ensName,
  ])
  ↓
AgentDashboard に OnChainProof セクションを表示
  ↓
[INSERT COIN FOR FIRST TRADE] ボタン (user gesture 必須)
  → uniswap.executeFirstSwap(walletClient) → txHash
  ↓
swap tx hash を OnChainProof に追加
```

## 想定リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| 0G Storage SDK のブラウザ動作不安定 | 0G Storage 賞ゼロ | 失敗時は data URI に fallback (CID は null)。SDK debug を Plan.md にログ |
| Sepolia ENS の `gradiusweb3.eth` 取得失敗 | ENS 賞ゼロ | Hackathon 専用 namespace 取得、もしくは ENS DAO の Sepolia faucet 利用 |
| Uniswap Sepolia faucet 切れ | swap 実行不能 | demo 動画用に事前に WETH / USDC を agent wallet に充填しておく |
| 0G Galileo RPC が rate limit | mint 失敗 | 自前 RPC 設定、もしくは Galileo public RPC を複数登録 |
| iNFT が ERC-7857 厳密準拠でない | 0G iNFT 賞減点 | OpenZeppelin の ERC-721 + tokenURI で「ERC-7857-style」を主張、demo で intelligence embedded を強調 |
